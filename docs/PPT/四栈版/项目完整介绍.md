# 流域水环境 AI 智能监测与预警平台 — 完整项目介绍

## 一、项目概述

本项目是一套面向流域水环境的 **"端-边-云-智"一体化智能监测与预警平台**，从前端水质感知硬件，到边缘采集与联网传输，再到云端多模态数据存储、AI 推理与可视化大屏，形成闭环。平台聚焦水文实时监测、污染源追溯、智能告警与决策辅助，可适配不同流域与监测区域的部署需求。

整体架构分为 **四层**：

```
感知层(硬件) → 传输层(MQTT) → 平台层(多模态存储+AI引擎) → 应用层(Web可视化)
```

---

## 二、硬件层 — 端侧感知与采集终端

### 2.1 多参数水质传感器阵列（可选配置）

> **重要说明**：水质传感器在终端硬件上为 **可选模块**，并非必备。下列"常规五参数"为**基础版出厂搭配**，其余参数（浊度、溶解氧、氨氮、COD、余氯、叶绿素 a、蓝绿藻、ORP、水位、流速等）为**可选扩展**，可按项目需求自由增减。平台软件侧已对多类水质指标做通用化建模，传感器新增/替换仅需在站点元数据与接入配置中声明，即可自动进入时序库、告警与 AI 分析链路。

**2.1.1 基础版五参数（默认搭配）**

| 参数 | 量程 | 测量精度 | 用途 |
|---|---|---|---|
| **pH 值** | 0 ~ 14 | ±0.02 | 酸碱度异常识别 |
| **电导率 EC** | 0 ~ 2000 μS/cm | ±1% FS | 溶解性离子浓度 |
| **水温度** | -40 ~ 125 °C | — | 热污染/季节基线 |
| **总溶解固体 TDS** | 0 ~ 1000 ppm | — | 矿化度/浊度关联分析 |
| **盐度** | 0 ~ 1300 ppm | — | 咸潮/工业排放追溯 |

**2.1.2 可选扩展参数**

- 浊度、溶解氧 (DO)、氧化还原电位 (ORP)
- 氨氮 (NH₃-N)、化学需氧量 (COD)、总氮/总磷 (TN/TP)
- 余氯、叶绿素 a、蓝绿藻
- 水位、流速、流量（水文物理量）

基础五参数即可驱动平台的规则告警、时序异常检测与图谱污染源追溯模型；扩展参数接入后，AI 引擎会自动纳入多源归因与决策建议。

### 2.2 PCB 核心板（主控 MCU，标配）

- **主控芯片**：STM32F407VET6
  - 内核：ARM 32-bit Cortex-M4（含单精度 FPU 浮点单元）
  - 主频：168 MHz，性能 210 DMIPS / 566 CoreMark
  - 存储：Flash 512 KB，SRAM 192 KB
  - 工作电压：1.8 V ~ 3.6 V
  - 封装：LQFP-100 (14×14×1.4 mm)
- **双路供电**：
  1. 24 V DC 电源适配器输入（常电）
  2. 18 V 太阳能板输入（离网/补能）
- **选型动因**：M4 带 FPU，能在 MCU 本地完成传感器数据的滑动平均、温补、卡尔曼滤波等预处理，再以结构化 JSON 上行，有效降低云端解析压力。

### 2.3 太阳能供电系统（标配，与 DC 适配器二选一）

| 项目 | 参数 |
|---|---|
| 输出功率 | 50 W |
| 输出电压 | 18 V |
| 峰值电流 | 2.55 A |
| 开路电压 | 24.3 V |
| 短路电流 | 2.7 A |
| 尺寸 | 670 × 430 × 25 mm |

满足野外无电源环境下 **24×7 不间断监测**，配合备用锂电池实现阴雨天续航。

### 2.4 网络通信模块（标配）

- **以太网 PHY**：LAN8720A-CP-TR
  - 单通道 10/100BASE-T/TX 以太网收发器
  - 兼容 IEEE 802.3 / 802.3u / 802.3-2005
  - 10 Mbps / 100 Mbps 自适应
  - RMII 接口，与 STM32F407 的 MAC 无缝对接
- **上行协议**：MQTT over TCP/IP，主题按 `station/{station_id}/metrics` 组织，由平台侧 EMQX 接入。

## 三、传输层 — MQTT 设备接入

- **消息中间件**：EMQX v5.5.1（MQTT Broker）
- **设备接入模式**：终端通过以太网 PHY 接入局域网/工业专网，以 MQTT 客户端身份上行。
- **平台订阅**：后端 `get_ingestion_service`（`backend/app/data/core/ingestion.py`）在应用生命周期启动时订阅 Broker，并支持**外部 MQTT 数据源**通过 `mqtt_connection` 动态管理与持久化，平台重启后自动恢复订阅。
- **入库路径**：原始报文 → 解析 → 写入 TDengine 时序库 → 触发流式规则与 AI 分析。

---

## 四、平台层 — 模块化单体后端

后端采用 **FastAPI 模块化单体架构**，统一入口 `backend/app/main.py` 通过 `lifespan` 协程依次初始化 Station / Alert / Report / Data / AI 五大模块，对外暴露 `/api/v1/*` 路由。

### 4.1 技术栈

| 层次 | 选型 |
|---|---|
| 应用框架 | FastAPI + Uvicorn（Python 3.11） |
| 时序库 | **TDengine v3.2.3.0**（水质指标高频写入） |
| 关系+空间+向量 | **PostgreSQL 14 + PostGIS + pgvector**（站点空间属性、知识向量） |
| 图数据库 | **Neo4j v5.15-community + APOC**（河网拓扑、污染源知识图谱） |
| 消息总线 | **EMQX v5.5.1**（设备接入与事件广播） |
| 对象存储 | **MinIO**（报表、模型、知识库文件） |
| 反向代理 | Nginx |

### 4.2 六大业务模块

| 模块 | 职责 | 核心存储 |
|---|---|---|
| **station** | 水文站/传感器元数据、行政区与空间属性 | PostgreSQL + PostGIS |
| **data** | MQTT 接入、时序写入、外部连接管理 | TDengine + EMQX |
| **alert** | 规则告警 + 模型告警 + 事件广播 | PostgreSQL + EMQX |
| **ai** | 多智能体集群推理 | TDengine + Neo4j + MinIO |
| **report** | PDF / Excel 报表生成，结果落 MinIO | MinIO + PostgreSQL |
| **gateway**（已并入 main） | 健康检查、统一 CORS、路由聚合 | — |

### 4.3 AI 引擎（核心亮点）

#### 4.3.1 多智能体集群（Multi-Agent）
位于 `backend/app/ai/agents/`，由 `CoordinatorAgent` 协调三类专职 Agent：

- **MonitorAgent** — 实时监听 TDengine 指标流，做滑窗异常检测
- **AnalysisAgent** — 调度时序/图谱/空间/知识四类引擎做多源归因
- **DecisionAgent** — 基于 `alert_levels` 与知识库生成处置建议与分级输出
- **OutputGenerator** — 结构化事件转自然语言报告

#### 4.3.2 四大推理引擎
位于 `backend/app/ai/engines/`：
- `time_series.py` — 时序异常检测（阈值 + 模型）
- `graph.py` — Neo4j 河网拓扑推理，上下游污染源追溯
- `knowledge.py` — 基于 pgvector/MinIO 的案例与治理方案检索
- （空间分析）— PostGIS 缓冲区、流域汇水区计算

#### 4.3.3 执行闭环
```
感知数据 → MonitorAgent 发现异常 → Coordinator 调度
    → AnalysisAgent 时序+图谱+空间+知识四维归因
    → DecisionAgent 给出处置方案
    → EMQX 广播 + Web 大屏告警 + PDF 报表
```

## 五、应用层 — Web 可视化大屏

前端基于 **React 19 + TypeScript + Vite 8**，定位为 *决策者驾驶舱*。

### 5.1 技术栈（参考 `frontend/package.json`）
- UI：Ant Design 6、Tailwind CSS 3、styled-components
- 地理可视化：**Three.js + @react-three/fiber + @react-three/drei**（无边界三维地形，默认视角与覆盖区域可配置，便于适配不同流域）、**deck.gl**、d3-geo、topojson-client、AMap 高德地图
- 数据可视化：ECharts 5 + echarts-gl
- 状态/路由：Zustand、React Router 7
- 动画/热力：GSAP、keli-heatmap

### 5.2 主要业务页面（`frontend/src/pages/`）
- **Home / Dashboard** — 首页大屏与综合仪表盘
- **Map** — 通用流域三维 GIS 大屏，默认视角、法线图层范围、路网等级等均可配置，不绑定特定地区，可按需切换到任意流域
- **Stations / StationDetail** — 站点管理与详情
- **MqttConnections / MqttData** — 外部 MQTT 数据源接入
- **Alerts / AlertDetail / AlertRules** — 告警中心与规则管理
- **Agents / Models / Prediction** — AI 多智能体与预测
- **Graph / RiverTopology / PollutionSources** — 河网拓扑与污染源图谱
- **Knowledge / CaseDetail** — 知识库与治理案例
- **Reports** — PDF / Excel 报表

## 七、技术亮点

1. **端云协同**：STM32F407 的 FPU 在端侧完成数字滤波，上行数据已过噪声处理；平台侧流式计算 + AI 仅处理高价值事件。
2. **离网可部署**：50 W 太阳能板 + 18 V 直供，适用于无市电野外站点。
3. **多模态融合**：**时序（TDengine）+ 空间（PostGIS）+ 图谱（Neo4j）+ 向量（pgvector）+ 对象（MinIO）** 五库协同，AI 分析兼具"快、准、可解释"。
4. **多智能体自治**：Coordinator + Monitor + Analysis + Decision 四 Agent 分工协作，支持模型热加载与动态调度。
5. **通用三维流域大屏**：基于 Three.js 自绘地形与河网拓扑，地图视角、覆盖区域与图层资源全部参数化，可在不同流域/项目间自由切换，不与具体地区耦合。
6. **模块化单体**：统一 FastAPI 入口，部署简单，同时保留业务模块边界，便于按需拆分微服务。

## 八、应用价值

- **监管侧**：实时掌握流域水质动态，快速定位污染源，提高执法效率。
- **应急侧**：AI 自动分级 + 处置建议输出，缩短突发水环境事件响应时间。
- **运营侧**：可离网部署的低功耗硬件 + 云端智能大脑，单位运维成本显著下降。
- **科研侧**：多模态历史数据资产沉淀，支撑水环境建模与长期趋势研究。
